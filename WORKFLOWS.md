# Interactive Workflows Guide

This document explains the interactive workflows and user journeys implemented in the Cold Storage Management System.

## 🎯 Overview

The application now features **fully interactive workflows** that allow users to:
- ✅ Take action on alerts (resolve, escalate)
- ✅ Approve or reject transfer requests
- ✅ Generate compliance documents
- ✅ Track action history with financial impact
- ✅ Experience real-world user scenarios

## 🔄 Workflow Features

### 1. Alert Management Workflow

#### **Facility Level** (Ravi - Facility Manager)

**Scenario: Temperature Breach Alert**

1. **Alert Appears**
   - Critical alert shows in red box at top
   - Shows: "Zone B temperature at -12°C (should be -18°C)"
   - Financial impact: ₹45,000 at risk

2. **User Clicks Alert**
   - Modal opens with full alert details
   - Shows all relevant information:
     - Time, Zone, Severity
     - Financial impact highlighted
     - Resolution notes text area

3. **User Actions Available:**
   - **Resolve Alert**: Enter notes and mark as resolved
     - Action logged to history
     - Financial impact recorded (₹45,000 saved)
     - Alert status changes to "RESOLVED"
     - Alert becomes grayed out

   - **Escalate to Regional**: If unable to resolve
     - Alert forwarded to Priya (Regional Manager)
     - Status changes to "ESCALATED"
     - Action logged with "pending" status

#### **Regional Level** (Priya - Regional Manager)

**Scenario: Escalated Alert from Facility**

1. **Receives Escalated Alert**
   - Shows blue badge: "ESCALATED TO REGIONAL"
   - Shows original alert details
   - Shows who escalated and when

2. **Regional Actions:**
   - Can resolve alert (with notes)
   - Can escalate to HQ if needed
   - All actions tracked with actor name

#### **HQ Level** (Rajesh - COO)

**Scenario: Escalated Alert from Region**

1. **Receives Escalated Alert**
   - Shows blue badge: "ESCALATED TO HQ"
   - Full escalation history visible

2. **HQ Actions:**
   - Resolve alert (final resolution)
   - Strategic decision making
   - All actions tracked

### 2. Transfer Approval Workflow

#### **Scenario: Cross-Facility Transfer Request**

**At Regional Dashboard (Priya):**

1. **View Transfer Suggestion**
   - AI suggests: "Move 5 tonnes Mangoes from Chennai to Bangalore"
   - Shows savings: ₹40,000
   - Shows reason: "High demand in Bangalore, expiring in 7 days"

2. **Actions Available:**
   - **Approve Transfer**:
     - Status changes to "APPROVED"
     - Action logged: "Approved transfer: Mangoes..."
     - Shows approved by: "Priya (Regional Manager)"
     - Financial impact: +₹40,000 saved
     - Transfer shows in history

   - **Reject Transfer**:
     - Status changes to "REJECTED"
     - Action logged with reason
     - No financial impact recorded

**At Facility Dashboard (Ravi):**

1. **View Pending Transfer**
   - Shows transfer request from regional manager
   - Can see details and savings
   - Status shows current state

### 3. Compliance Document Generation

#### **Scenario: Generating FSSAI Report**

**At Facility Dashboard (Ravi):**

1. **Click "Generate Report"**
   - Button available for each document type:
     - FSSAI
     - FDA
     - Temperature Log
     - Cleaning Log
     - Incident Report
     - EU GDP

2. **Report Generation**
   - System creates report with:
     - Report ID: `RPT-{timestamp}`
     - Facility: "Chennai"
     - Generated by: "Ravi (Facility Manager)"
     - Valid until: 30 days from generation
     - Status: "VALID"

3. **Action Logged**
   - Action type: "report_generated"
   - Description: "Generated FSSAI report for Chennai"
   - Actor: "Ravi (Facility Manager)"
   - Status: "completed"

---

### 4. Action History Tracking

#### **Viewing Action History**

**Toggle History Button:**
- Click "Show History" button in Facility Dashboard
- Shows chronological list of all actions

**Action Details Displayed:**
1. **Icon** - Type of action
   - ✓ Green: Resolved alerts, approved transfers
   - ✗ Red: Rejected transfers
   - 📄 Purple: Reports generated
   - 🔧 Orange: Maintenance requests

2. **Description** - What was done
   - "Resolved temperature alert: Zone B temperature..."
   - "Approved transfer: Mangoes from Chennai to Bangalore"
   - "Generated FSSAI report for Chennai"

3. **Financial Impact**
   - Green text: Savings (e.g., "Saved ₹45,000")
   - Red text: Costs
   - Shows monetary value of each action

4. **Metadata**
   - Actor: Who performed the action
   - Timestamp: When it happened
   - Facility: Which facility
   - Status: completed/pending

## 🎮 How to Use Interactive Features

### **Facility Dashboard (Chennai)**

1. **Open the application**: http://localhost:5173/
2. **Select role**: Choose "Facility Dashboard" from dropdown
3. **Try these workflows:**

   **a) Resolve an Alert:**
   - Click on any alert in the Critical Alerts section
   - Read the alert details in the modal
   - Enter resolution notes (e.g., "Compressor sensor replaced, temperature back to normal")
   - Click "Resolve Alert"
   - ✅ Alert marked as resolved
   - ✅ Action appears in history
   - ✅ Financial impact saved

   **b) Escalate an Alert:**
   - Click on an alert
   - Click "Escalate to Regional" button
   - ✅ Alert status changes to "ESCALATED"
   - ✅ Action logged
   - ✅ Alert shows escalation details

   **c) Approve Transfer:**
   - Scroll to "Cross-Facility Transfer Suggestions"
   - Click "Approve Transfer" button
   - ✅ Status changes to "APPROVED"
   - ✅ Action logged with savings
   - ✅ Shows in history

   **d) Generate Report:**
   - Scroll to "Compliance & Documentation"
   - Click "Generate Report" for any document type
   - ✅ Alert shows success
   - ✅ Action logged

   **e) View History:**
   - Click "Show History" button in header
   - ✅ See all actions with financial impact
   - ✅ Toggle to hide

## 📊 Data Flow Architecture

```
User Action (Ravi)
    ↓
WorkflowContext (State Management)
    ↓
Action Created & Logged
    ↓
State Updated (alerts, transfers, actions)
    ↓
UI Re-renders with New State
    ↓
History Updated
    ↓
Financial Impact Calculated
```

## 🚀 How It Works (Technical)

### **Context Provider Pattern**
```typescript
<WorkflowProvider>
  <App />
</WorkflowProvider>
```

### **State Management**
- `alerts`: Array of alert objects
- `transfers`: Array of transfer requests
- `actions`: Array of action history
- `reports`: Array of generated reports

### **Actions Available**
- `resolveAlert(alertId, notes, actor)`
- `escalateAlert(alertId, to, actor)`
- `approveTransfer(transferId, actor)`
- `rejectTransfer(transferId, actor)`
- `generateReport(type, facility, actor)`
- `getActionsByFacility(facility)`
- `getOpenAlerts(facility)`
