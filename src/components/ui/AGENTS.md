# UI PRIMITIVES

## OVERVIEW
shadcn/ui-style primitives. Hand-maintained — not auto-generated by shadcn CLI.

## COMPONENTS
| File | Exports | Notes |
|------|---------|-------|
| `card.tsx` | Card, CardHeader, CardContent, CardTitle, CardDescription | |
| `button.tsx` | Button | cva variants: default/outline/ghost/destructive/link |
| `badge.tsx` | Badge | cva variants: default/secondary/outline/destructive |
| `tabs.tsx` | Tabs, TabsList, TabsTrigger, TabsContent | Radix UI via shadcn |
| `dialog.tsx` | Dialog, DialogTrigger, DialogContent, DialogHeader, DialogTitle | |
| `select.tsx` | Select, SelectTrigger, SelectContent, SelectItem | |
| `map.tsx` | MapContainer | MapLibre wrapper primitive |

## CONVENTIONS
- Every primitive: `React.forwardRef` + `cn()` + `displayName`
- Use `cva` (class-variance-authority) for variant styling
- Extend via `className` prop — never hardcode overrides inside the primitive
- Import `cn` from `@/lib/utils`

## ANTI-PATTERNS
- Do NOT add business logic or data fetching to primitives
- Do NOT skip `displayName` — React DevTools depends on it
- Do NOT create new primitives without following the forwardRef + cn + displayName pattern
